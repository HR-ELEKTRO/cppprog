	Een \emph{abstracte} class is een class die zijn implementatie details verbergt voor de gebruiker.
	Omdat ook de compiler niets weet van de implementatie van een abstracte class, ook de benodigde geheugenruimte niet, kan een object van een abstracte class alleen dynamisch worden aangemaakt met behulp van |new|, zie \csecref{sec:new}.
	Je hebt dus altijd een pointer of een reference, zie \csecref{reference}, nodig om te verwijzen naar een object van een abstracte class.
	Een abstracte class heeft minstens één virtuele functie, zie \csecref{virtual}.
	De abstracte class vormt de interface tot de concrete class die deze interface implementeert.
	Een abstracte class die de interface verzorgt voor een aantal verwante classes noemen we een \emph{\USE{polymorphic type}}.
\end{itemize}

\subsection{Templates}
\begin{itemize}
	\item 
	In \Cpp{}20 worden zogenoemde \emph{Concepts} geïntroduceerd.
	Een concept is een predicate voor een type.
	Een predicate is een bewerking die waar of onwaar is.
	Met behulp van concepts kunnen je template argumenten beperken tot verschillende types.
	Bijvoorbeeld: |template<Elem T>| kan gebruikt worden voor types |T| waarvoor geldt dat |Elem<T>| waar is.
	Je kunt een concept dus ook zien als een soort van type van een type (dit wordt ook wel een metatype genoemd).
	\item 
	%TODO spelling consequent maken
	Een member-functie kan een functie-template zijn maar dat geldt niet voor een virtual functie.		
	
	\item 
	Een functieobject wordt ook wel een \emph{policy object} genoemd.
	\item 
	Een lambda kan handig gebruikt worden om een groep statements te veranderen in een expressie.
	%TODO: kan dit gebruikt worden om een Breuk in te lezen in operator<<?
	Bijvoorbeeld bij een ingewikkelde initialisatie:
	\begin{cpplstfragment}
	vector<int> v = [&] {
		switch (m) {
			case 0:
			return {0, 2, 4, 6, 8};
			case 1:
			return {1, 3, 5, 7, 9};
			default:
			return {};
		}
	};
	\end{cpplstfragment}
	
	\item 
	Ook een variabele kan een template zijn zodat je ook variabelen en constanten als template kunt definiëren.
	%TODO: duidelijk use case ontbreekt.
	\begin{cpplstfragment}
	template<typename T> constexpr T pi = atan(static_cast<T>(1))*4;
	\end{cpplstfragment}
	Je kunt nu de constante pi als volgt gebruiken:
	\begin{cpplstfragment}
	auto d{pi<double>}; // d is double
	auto f{pi<float>}; // f is float
	\end{cpplstfragment}
	Zie \proglink{pi.cpp}.
	
	\item 
	Met |using| kun je een parametertype van een template \squote{invullen}.
	In het Engels wordt dit \emph{argument binding} genoemd. 
	\begin{cpplstfragment}
	template<typename Value>
	using String_map = map<string, Value>;
	
	String_map<int> m; // m is van het type map<string, int>
	\end{cpplstfragment}

\subsection{Containers}
\begin{itemize}

	|vector<int> v1{10}| is een vector met één element met de waarde 10.

	\item 
	Als een |vector| groeit kunnen de elementen van plaats veranderen. 
	Pointers en iteratoren naar deze elementen staan dan naar verkeerde adressen te wijzen.
	Dit kun je voorkomen door met behulp van de functie |reserve|, van tevoren, voldoende elementen te reserveren.
	Zie \proglink{vector_reserve.cpp}.
	\item 
	|Save_vector| is een mooi voorbeeld van overerving.
	Zie \proglink{Save_vector.cpp}.
	Werkt niet polymorf omdat operator[] niet |virtual| gedefinieerd is.
	\item 
	Range checking kan soms ook aangezet worden als compiler optie.
	In gcc is dat de optie |-D_GLIBCXX_DEBUG|.
	\item 
	|count| bij |list| is $O(1)$ maar |count| bij |forward_list| is $O(n)$.	 
	\item 
	De standaardcontainer hebben zogenoemde \emph{emplace}-operaties.
	Een |vector| heeft naast de bekende |push_back|-operatie ook een |emplace_back|-operatie.
	De |emplace_back| combineert het toevoegen van een element in de vector met het aanroepen van de constructor van dit element.
	Dit voorkomt het maken van een kopietje.
	In het programmafragment:
	\begin{cpplstfragment}
	vector<Breuk> v;
	v.push_back(Breuk{1, 2});
	\end{cpplstfragment}
	Kan de laatste regel efficiënter gecodeerd worden als:
	\begin{cpplstfragment}
	v.emplace_back(1, 2);
	\end{cpplstfragment}
	Zie \proglink{vector_emplace}.	
\end{itemize}

\subsection{Utilities}
\begin{itemize}
	\item 
	Sinds \Cpp{}11 bevat de standaard library zogenoemde \emph{smart pointers}.
	Een |unique_ptr| is een verwijzing naar een object.
	Deze pointer beheert de levensduur van dit object en heeft een move constructor \csecref{sec:move} waarmee het beheer overgedragen kan worden aan een andere |unique_pointer|.
	Een |unique_pointer| kan, zoals de naam al suggereert, \emph{niet} gekopieerd worden.
	Hierdoor kun je een |unique_pointer| teruggeven vanuit een functie.
	Het object wordt verwijderd als de |unique_pointer| die het object beheerd wordt verwijderd.
	Zie \proglink{unique_pointer.cpp}.
	\item 
	Een |shared_ptr| is ook een verwijzing naar een object.
	Deze pointer beheert ook de levensduur van het betreffende object.
	Een |shared_pointer| kan in tegenstelling tot een |unique_pointer| wel gekopieerde worden.
	Het object wordt pas verwijderd, als de laatste |shared_pointer| die naar dit object verwijst, wordt verwijderd.
	De standaard library bevat in de include file |memory| de functies |make_shared()| en |make_unique| waarmee objecten dynamisch aangemaakt kunnen worden zonder dat |new| expliciet wordt aangeroepen.
	Smart pointers zijn te verkiezen boven gewone pointers omdat ze de kans op memory leaks sterk verkleinen.
	Maar smart pointers blijven een soort pointer, dus gebruik ze alleen als er geen eenvoudigere oplossing mogelijk is.
	Vaak kan het gebruik van een standaard container het gebruik van pointers voorkomen.
	Als je polymorfisme gebruikt dan moet je een pointer gebruiken.
	Het gebruik van een |unique_pointer| ligt dan voor de hand.
	Als het polymorfe object gedeeld moet worden, heb je een |shared_pointer| nodig.
	%TODO: plaatsen opzoeken waar new gebuikt wordt en verwijzen naar gebruik smart pointers.
	\item 
	De standaard functie |forward| lijkt op de eenvoudigere |move| en kan gebruikt worden om een verzameling argumenten exact door te geven.
	Deze functie wordt onder andere gebruikt om |make_unique| te implementeren:
	\begin{cpplstfragment}
template<typename T, typename ... Args>
unique_ptr<T> make_unique(Args&&... args) {
	return unique_ptr<T>{new T(std::forward<Args>(args)...)}
}
	\end{cpplstfragment}
	\item 
	Sinds Cpp{}20 bevat de standaard library de class |span|.
	Dit is de combinatie van een pointer en een offset.
	\item 
	De |valarray<T>| is een speciale container waarbij T een numeriek type moet zijn.
	Er kunnen rekenkundige bewerkingen op dit type uitgevoerd worden.
	Een |valarray| lijkt dus meer op een vector zoals we die kennen uit de wiskunde dan een |vector|.
	Zie \proglink{valarray.cpp}. 
	Deze rekenkundige bewerkingen kunnen efficiënt geïmplementeerd worden.
	Zo kan de bewerking:
	|v3 = 2*v1 + v2| door de compiler vertaald worden in één loop waarin voor elke element van |v| de volgende code wordt uitgevoerd: |v3[i] = 2*v1[i] + v2[i];|.
	\item 
	Andere utility classes: |bitset|, |pair|, |tuple|, |variant|, |optional| en |any|.
	|pair| en |tuple| zijn heterogeen alle andere containers zijn homogeen.
	|bitset| bevat een vast aantal bits.
	|variant| is een beter alternatief voor |union|.
	|optional| is een object dat mogelijk leeg is.
	|any| is een object dat elk willekeurig type kan bevatten.
	\item 
	Type deduction heeft de functie |make_pair| sinds \Cpp{}17 min of meer overbodig gemaakt.
	%TODO zoek waar |make_pair| gebruikt wordt.
	Zie \proglink{make_pair.cpp}.
	Hetzelfde geldt voor |make_tuple|.  
	\item 
	De \Cpp{}17 standaard library bevat een pool allocator.
	Deze allocator is sneller dan de standaard allocator, als je veel objecten van dezelfde grootte nodig hebt.
	Alle standaard containers hebben een extra template argument waarmee je de allocator kunt opgeven.
	\item 
	De include file |<chrono>| bevat utilities om met tijd om te gaan.
	Er zijn user defined suffixes gedefinieerd die je kunt gebruiken om je code leesbaarder te maken.
	Bijvoorbeeld:
	\begin{cpplstfragment}
	this_thread::sleep(10ms+500us);
	\end{cpplstfragment}
	Ook kun je eenvoudig een timer maken om de executietijd van een stuk code te meten.
	\begin{cpplstfragment}
	auto start {chrono::high_resolution_clock::now()};
	// code die je wilt timen
	auto stop {chrono::high_resolution_clock::now()};
	auto duration {chrono::duration_cast<std::chrono::microseconds>(stop - start).count()};
	cout << "Executietijd = " << duration << " us\n";
	\end{cpplstfragment}
	\item 
	Het kiezen van een implementatie afhankelijk van het type van de iterator zal in \Cpp{}20 veel eenvoudiger worden door het gebruik van concepts.
\end{itemize}

\subsection{History and Compatibility}
\begin{itemize}
	\item 
	Attributes zijn een gestandaardiseerde manier om taaluitbreidingen te implementeren in \Cpp{}.
	Een attribute heeft de volgende syntax |[[naam]]|.
	Er zijn ook een aantal standaard attributes gedefinieerd.
	Zie \url{https://en.cppreference.com/w/cpp/language/attributes}.
	Hier volgen enkele voorbeelden:
	\begin{itemize}
		\item 
		De hoofdfunctie in veel embedded systemen bevat een |while(1)|-loop omdat deze functie \squote{altijd} moet blijven draaien.
		Sommige compilers geven in dit geval een warning.   
		Je kunt aangeven dat het juist de bedoeling is dat de functie nooit terugkeert door de functie te voorzien van het attribuut |[[noreturn]]|.
		\item 
		In |\switch|-statements gebruik je meestal aan het einde van elke |case| een |break|-statement.
		Soms wil je echter dat het programma na uit uitvoeren van de |case| verdergaat met het uitvoeren van de volgende |case|.
		In dit geval eindigt de |case| niet met een |break|.
		Veel compilers geven in dit geval een warning.
		Je kunt aangeven dat je bewust de |break| hebt weggelaten door het attribuut |[[fallthrough]]| te gebruiken.
		Zie \proglink{attributes.cpp}.
	\end{itemize}
	\item 
	De standaard library bevat ook de |ratio| voor \emph{compile-time rational arithmetic}.
	De teller en de noemer zijn template parameters.
	Een duidelijk use case ontbreekt.
\end{itemize}
