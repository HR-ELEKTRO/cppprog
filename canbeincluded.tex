	\item
	Als een klasse X een destructor heeft die een niet-triviale taak uitvoert, zoals het vrijgeven van geheugen of een vergrendeling, heeft de klasse waarschijnlijk de volgende speciale memberfuncties nodig:
\begin{cpplstfragment}
class X {
public:
	X(Sometype); // constructor
	X(); // default constructor
	X(const X&); // copy constructor
	X(X&&); // move constructor
	X& operator=(const X&); // copy assignment
	X& operator=(X&&); // move assignment
	~X(); // destructor
	// ...
};
\end{cpplstfragment}
	\item
	Behalve de \squote{gewone} constructor worden deze speciale memberfuncties door de compiler gegenereerd wanneer dat nodig is.
	Als je expliciet wilt aangeven dat er standaardimplementaties moeten worden gegenereerd kun je de syntax |=default| gebruiken.
	Als je voor bepaalde speciale memberfuncties expliciet bent, worden andere speciale memberfuncties niet gegenereerd.
	Als je expliciet wilt aangeven dat er geen standaardimplementatie gegenereerd mag worden, kun je de syntax |=delete| gebruiken.
	\item

	Dit heet in het Engels \dquote{\USE{default member initializers}}.
\end{itemize}

\subsection{Concepts and Generic Programming}
\begin{itemize}
	\item
	Een concept is een predicaat dat waar of onwaar kan zijn voor een bepaald type.
	Er zijn een aantal standaard concepts gedefinieerd in de standaard library, zie \url{https://en.cppreference.com/w/cpp/concepts.html}.
	Een concept kan als template parameter worden gebruikt om een template te beperken tot bepaalde types.
	Dit zorgt ervoor dat de compiler betere foutmeldingen kan geven als een template wordt gebruikt met een type dat niet aan de eisen van het concept voldoet.
	Zie \proglink{concepts.cpp}.
	Gebruik bij voorkeur deze standaard concepts in plaats van zelf gedefinieerde concepts.
	Je kunt een concept ook zelf definiëren, bijvoorbeeld als combinatie van andere concepts.
	Zie \proglink{concepts.cpp}.
	Bijvoorbeeld het concept |Number| kan gedefinieerd worden als combinatie van de standaard concepts |integral| en |floating_point| waarbij de character types worden uitgesloten:
\begin{cpplstfragment}
template <typename T>
concept Number = (std::integral<T> || std::floating_point<T>) 
                 && !std::same_as<T, bool>
                 && !std::same_as<T, char>
                 && !std::same_as<T, unsigned char>
                 && !std::same_as<T, char8_t>
                 && !std::same_as<T, char16_t>
                 && !std::same_as<T, char32_t>
                 && !std::same_as<T, wchar_t>;

auto add(Number auto a, Number auto b) {
    return a + b;
}
\end{cpplstfragment}	
	\item

	Zie \proglink{require.cpp}.
	\item
	De concepten die voor een template zijn gespecificeerd, worden gebruikt om argumenten te controleren op het moment dat het template wordt gebruikt.
	Ze worden niet gebruikt om het gebruik van de parameters in de definitie van het template te controleren.
	\item
	Door een |auto|-parameter te gebruiken, wordt een functie een functietemplate.
	Zie \proglink{swap_auto.cpp}.
	Maar pas op nu kun je ook een |int| met een |double| uitwisselen, wat waarschijnlijk niet de bedoeling is.
	Dat kan niet bij een template waarbij beide parameters dezelfde template parameter hebben.
	Zie \proglink{swap_template.cpp}.

	Een |strong_ordering| geeft aan dat de twee objecten die vergeleken worden volledig geordend zijn, dat wil zeggen dat ze ofwel gelijk |strong_ordering::equal| or |strong_ordening::equivalent| zijn, orwel dat het ene object kleiner |strong_ordering::less| or groter |strong_ordering::greater| is dan het andere.
	Een |partial_ordering| heeft de volgende mogelijke waarden: |partial_ordering::less|, |partial_ordering::greater|, |partial_ordering::equivalent| of |partial_ordering::unordered|.
	Twee objecten zijn |partial_orderingunordered| als ze niet geordend kunnen worden, bijvoorbeeld als een object van het type |double| de waarde NaN\footnote{%
		Zie \url{https://en.wikipedia.org/wiki/NaN}.
	} heeft.
	Twee objecten |a| en |b| zijn |partial_ordering::equivalent| als ze niet |partial_ordering::unordered| zijn en als |a| niet kleiner en ook niet groter is dan |b|.
	Een |weak_ordering| heeft de volgende mogelijke waarden: |partial_ordering::less|, |partial_ordering::greater| of |partial_ordering::equivalent|.
	Twee objecten die |weak_ordering::equivalent| zijn, zijn niet noodzakelijkerwijs gelijk, maar worden wel als equivalent beschouwd voor de doeleinden van sorteren en vergelijken.
	Als je bijvoorbeeld objecten van de class |string| case insensitive vergelijkt, dan kunnen twee strings die verschillen in hoofdletters en kleine letters als |weak_ordering::equivalent| worden beschouwd, ook al zijn ze niet exact gelijk.

	De drie verschillende soorten ordeningen ondersteunen bepaalde impliciete conversies. Een |strong_ordering| kan impliciet worden geconverteerd naar een |partial_ordering| of een |weak_ordering|. Een |weak_ordering| kan impliciet worden geconverteerd naar een |partial_ordering|.
