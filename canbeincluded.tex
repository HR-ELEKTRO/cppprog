	\begin{cpplstfragment}
void f() noexcept {

}

\subsection{Classes}
\begin{itemize}
	\item 
	Eenvoudige versie van |Breuk| maken die gebruik maakt van copy by value in plaats van references.
	Zie |Breuk4.cpp|.
	\item
	Een \emph{abstracte} class is een class die zijn implementatie details verbergt voor de gebruiker.
	Omdat ook de compiler niets weet van de implementatie van een abstracte class, ook de benodigde geheugenruimte niet, kan een object van een abstracte class alleen dynamisch worden aangemaakt met behulp van |new|, zie \csecref{sec:new}.
	Je hebt dus altijd een pointer of een reference, zie \csecref{reference}, nodig om te verwijzen naar een object van een abstracte class.
	Een abstracte class heeft minstens één virtuele functie, zie \csecref{virtual}.
	De abstracte class vormt de interface tot de concrete class die deze interface implementeert.
	Een abstracte class die de interface verzorgt voor een aantal verwante classes noemen we een \emph{\USE{polymorphic type}}.
\end{itemize}

\subsection{Templates}
\begin{itemize}
	\item 
	In \Cpp{}20 worden zogenoemde \emph{Concepts} geïntroduceerd.
	Een concept is een predicate voor een type.
	Een predicate is een bewerking die waar of onwaar is.
	Met behulp van concepts kunnen je template argumenten beperken tot verschillende types.
	Bijvoorbeeld: |template<Elem T>| kan gebruikt worden voor types |T| waarvoor geldt dat |Elem<T>| waar is.
	Je kunt een concept dus ook zien als een soort van type van een type (dit wordt ook wel een metatype genoemd).
	\item 
	%TODO spelling consequent maken
	Een member-functie kan een functie-template zijn maar dat geldt niet voor een virtual functie.		
	
	\item 
	Een functieobject wordt ook wel een \emph{policy object} genoemd.
	\item 
	Een lambda kan handig gebruikt worden om een groep statements te veranderen in een expressie.
	%TODO: kan dit gebruikt worden om een Breuk in te lezen in operator<<?
	Bijvoorbeeld bij een ingewikkelde initialisatie:

		switch (m) {
			case 0:
			return {0, 2, 4, 6, 8};
			case 1:
			return {1, 3, 5, 7, 9};
			default:
			return {};
		}
	};
	\end{cpplstfragment}
\end{itemize}

\subsection{Utilities}
\begin{itemize}
	\item 
	De standaard functie |forward| lijkt op de eenvoudigere |move| en kan gebruikt worden om een verzameling argumenten exact door te geven.
	Deze functie wordt onder andere gebruikt om |make_unique| te implementeren:
	\begin{cpplstfragment}
template<typename T, typename ... Args>
unique_ptr<T> make_unique(Args&&... args) {
	return unique_ptr<T>{new T(std::forward<Args>(args)...)}
}
	\end{cpplstfragment}
	\item 
	Sinds Cpp{}20 bevat de standaard library de class |span|.
	Dit is de combinatie van een pointer en een offset.
	\item 
	Andere utility classes: |bitset|, |pair|, |tuple|, |variant|, |optional| en |any|.
	|pair| en |tuple| zijn heterogeen alle andere containers zijn homogeen.
	|bitset| bevat een vast aantal bits.
	|variant| is een beter alternatief voor |union|.
	|optional| is een object dat mogelijk leeg is.
	|any| is een object dat elk willekeurig type kan bevatten.
	\item 
	Type deduction heeft de functie |make_pair| sinds \Cpp{}17 min of meer overbodig gemaakt.
	Zie \proglink{make_pair.cpp}.
	Hetzelfde geldt voor |make_tuple|.  
	\item 
	De \Cpp{}17 standaard library bevat een pool allocator.
	Deze allocator is sneller dan de standaard allocator, als je veel objecten van dezelfde grootte nodig hebt.
	Alle standaard containers hebben een extra template argument waarmee je de allocator kunt opgeven.
	\item 
	De include file |<chrono>| bevat utilities om met tijd om te gaan.
	Er zijn user defined suffixes gedefinieerd die je kunt gebruiken om je code leesbaarder te maken.
	Bijvoorbeeld:
	\begin{cpplstfragment}
	this_thread::sleep(10ms+500us);
	\end{cpplstfragment}
	\item 
	Het kiezen van een implementatie afhankelijk van het type van de iterator zal in \Cpp{}20 veel eenvoudiger worden door het gebruik van concepts.
\end{itemize}

\subsection{Numerics}
\begin{itemize}
	\item 
	Als er een fout optreedt in een functie uit |<cmath>|, dan wordt dit gerapporteerd door de globale variabele |errno| te vullen met een foutcode.
	\begin{cpplstfragment}
	errno = 0;
	double d {sqrt(-1)};
	if (errno == EDOM) {
		cerr << "Domain error for sqrt(-1)\n";
	}
	\end{cpplstfragment}
	\item 
	De include file |<numeric>| definieert een aantal rekenkundige functies.
	Onder andere |inner_product|, |gcd| en |lcm|.
	|gcd| en |lcm| zijn toegevoegd in \Cpp{}17.
	\item 
	De minimale en maximale waardes van de ingebouwde types kunnen opgevraagd worden met functies die gedeclareerd zijn in |<limits>|.
	Bijvoorbeeld de maximale waarde voor een |int| is |numeric_limits<int>::max()|.
\end{itemize}

\subsection{History and Compatibility}
\begin{itemize}
	\item 
	Attributes zijn een gestandaardiseerde manier om taaluitbreidingen te implementeren in \Cpp{}.
	Een attribute heeft de volgende syntax |[[naam]]|.
	Er zijn ook een aantal standaard attributes gedefinieerd.
	Zie \url{https://en.cppreference.com/w/cpp/language/attributes}.
	Hier volgen enkele voorbeelden:
	\begin{itemize}
		\item 
		De hoofdfunctie in veel embedded systemen bevat een |while(1)|-loop omdat deze functie \squote{altijd} moet blijven draaien.
		Sommige compilers geven in dit geval een warning.   
		Je kunt aangeven dat het juist de bedoeling is dat de functie nooit terugkeert door de functie te voorzien van het attribuut |[[noreturn]]|.
		\item 
		In |\switch|-statements gebruik je meestal aan het einde van elke |case| een |break|-statement.
		Soms wil je echter dat het programma na uit uitvoeren van de |case| verdergaat met het uitvoeren van de volgende |case|.
		In dit geval eindigt de |case| niet met een |break|.
		Veel compilers geven in dit geval een warning.
		Je kunt aangeven dat je bewust de |break| hebt weggelaten door het attribuut |[[fallthrough]]| te gebruiken.
		Zie \proglink{attributes.cpp}.
	\end{itemize}
	\item 
	De standaard library bevat ook de |ratio| voor \emph{compile-time rational arithmetic}.
	De teller en de noemer zijn template parameters.
	Een duidelijk use case ontbreekt.
\end{itemize}
